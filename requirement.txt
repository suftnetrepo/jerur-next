Development Freeze – E2E Testing Phase

Do not implement any new product features.

The objective is to validate the Attendance and Pastoral Care modules using Playwright.

If a test fails because of a bug introduced during Phase 1 or Phase 2, fix the bug before continuing.

Do not redesign the application.

Do not add new functionality.

Only fix issues required for the tests to pass.

------------------------------------------------

Task 1

Review the existing Playwright test structure and reuse the current testing patterns.

Do not create a new testing architecture.

------------------------------------------------

Task 2

Create comprehensive Playwright E2E tests for the following modules.

Attendance

Test:

- Attendance page loads
- Service selector loads
- KPI cards display
- Service selection refreshes dashboard
- Status chips display correct counts
- Clicking status chips filters the table
- Search works
- Pagination works
- Member detail opens
- Care Signal renders correctly
- Care Case column renders correctly

------------------------------------------------

Attendance Submission

Test every attendance status.

- Present
- Online
- Absent
- Sick
- Travelling
- Working
- Family Commitment
- Needs Prayer
- Other

Verify:

- Submission succeeds
- Duplicate submission prevented
- Dashboard updates
- Statistics update

------------------------------------------------

Pastoral Care

Test:

- Dashboard loads
- KPI cards display
- Filters work
- Search works
- Pagination works
- Open Case
- Update Status
- Update Notes
- Assigned To filter
- Priority filter

------------------------------------------------

Care Case

Test:

Create Care Case

Verify:

- Drawer opens
- Validation works
- Save succeeds
- Table refreshes
- Status updates

------------------------------------------------

Regression

Verify existing modules still work.

- Login
- Members
- Services
- Dashboard
- Sermons

------------------------------------------------

Task 3

Run the complete Playwright suite.

Fix any bugs that prevent the tests from passing.

Only fix defects.

Do not introduce new features.

------------------------------------------------

For each bug include:

- Description
- Root cause
- Files changed
- Fix applied

5. Final pass rate

6. Screenshots captured

7. Videos captured (if configured)

8. Remaining issues

9. Recommendations

10. Overall assessment

------------------------------------------------

Goal

The project should reach a stable state with a high E2E pass rate before Phase 3 begins.

Do not start Phase 3 after completing this task.
Wait for review and approval.
