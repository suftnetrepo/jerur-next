# Attendance Summary Export Report

## Files Modified

- `app/protected/church/attendance/page.jsx`
- `utils/attendanceExport.js`
- `package.json`
- `package-lock.json`

## PDF Layout

### Title
- Attendance Summary Report

### Header
Includes:
- Church Name
- Service Name
- Service Date
- Generated Date & Time
- Generated By

### Executive Summary
A calculated summary paragraph is shown below the header.

Example output format:
- `108 of 120 members submitted attendance (90%). 8 members require pastoral attention.`

### Attendance Summary
Rendered as a two-column table with:
- Expected Members
- Attendance Submitted
- Attendance Rate
- Members Requiring Attention
- Open Care Cases

### Attendance Breakdown
Rendered as a two-column table:
- Status
- Count

Statuses included:
- Present
- Online
- Absent
- Sick
- Needs Prayer
- Travelling
- Working
- Family Commitment
- Other

### Members Requiring Pastoral Attention
Rendered as a leadership-safe table with only:
- Member
- Attendance Status
- Member Response / Reason

Rows included only when:
- Attendance status is `ABSENT`, `SICK`, or `NEEDS_PRAYER`
- or `wantsPastorContact === true`

### Footer
Includes:
- `Generated by Jerur Church Management System`
- `Confidential - For Church Leadership Use Only`

## Data Included

Included in the PDF:
- Current church name from the existing church endpoint
- Current selected service title
- Current selected service date filter
- Current authenticated user name/email for `Generated By`
- Current attendance dashboard KPI aggregates
- Current filtered attendance records for the selected service/date/search/status view

Explicitly excluded from the PDF:
- Full attendance register export
- Care signal
- Care case status
- Case owner
- Internal notes
- Priority and other operational case-management fields

## Aggregate Calculations

### Attendance Submitted
Calculated from the current dashboard KPI when available, otherwise from the filtered attendance export dataset.

### Attendance Rate
Calculated as:
- `Attendance Submitted / Expected Members * 100`

Rounded to the nearest whole percent.

### Members Requiring Attention
Calculated from the filtered attendance dataset where either:
- status is `ABSENT`
- status is `SICK`
- status is `NEEDS_PRAYER`
- or `wantsPastorContact` is `true`

### Open Care Cases
Read from the current dashboard KPI when available, otherwise derived from filtered rows with open care cases.

## Export Flow

- The dashboard now has a single button: `Attendance Summary`
- Clicking the button generates the PDF immediately
- Existing service/date/search/status filters are preserved and reused
- The export fetches the full filtered attendance set instead of only the current visible table page
- No attendance model changes were required
- No attendance API changes were required
